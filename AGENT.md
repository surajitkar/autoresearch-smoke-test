# Agent workflow (before every PR)

This repository uses **Agent Prompt Autoresearch**: each PR is attributed to an instruction **variant** so you can measure which prompts produce better reviews and CI outcomes.

## Steps (canonical â€” other agent docs only point here)

1. **Assign a variant** from the **repository root** (the directory that contains `.repo-autoresearch/`):

   ```bash
   python scripts/get_variant.py --task "<ticket-ref-or-description>"
   ```

   Examples:

   ```bash
   python scripts/get_variant.py --task "PROJ-142"
   python scripts/get_variant.py --task "LIN-55"
   python scripts/get_variant.py --task "add discount code to checkout"
   ```

2. **Read** `.repo-autoresearch/autoresearch_instructions.md` and follow it while coding. That file is generated by `get_variant`; it contains the active slice from **`.repo-autoresearch/program.md`** (per-variant `<!-- VARIANT: â€¦ -->` sections), or a fallback under `variants/` if configured.

3. **Open the PR** with the **tracking tag** printed by `get_variant` at the end of the **PR body** (required for attribution). Example shape:

   ```
   [autoresearch:task=PROJ-142:variant=compact_diff_v1]
   ```

   Without this tag, the PR cannot be attributed to a variant and experiment data is lost.

4. Wait for CI and review. The autoresearch workflow posts an evidence comment on the PR.

## First-time setup in a new repo

```bash
pip install agent-prompt-autoresearch
autoresearch-init --with-workflow
```

Configure GitHub Actions secrets `GIST_ID` and `GIST_TOKEN`, then commit `.repo-autoresearch/`, `scripts/`, and `.github/workflows/`.

## Not the same as karpathy/autoresearch

That project runs **LLM training** experiments. This framework runs **PR instruction** A/B tests. The shared idea is only: *iterate instructions and measure outcomes.*
